# Contributing to AI Tool Documentation Evaluation

Thank you for your interest in contributing! This project evaluates a wide range of AI tools that help generate documentation for codebases. The outputs support developers, support teams, and product implementers working with open-source and enterprise systems.

We welcome new tools, updated experiences, improved test outputs, and anything that helps broaden this knowledge base.

## 🚀 How to Contribute

### 1. Suggest a New Tool
Open an [issue](https://github.com/lextego/ai_tool_tests/issues) with:
- Tool name and URL
- Short description (AI focus, relevant use cases)
- Any early thoughts on its applicability

### 2. Add Test Outputs
To contribute results from your own testing:
1. Fork this repository
2. Create a new folder under `/` named after the tool (e.g. `/mytool`)
3. Add files such as:
    - `README.md` – Your write-up or notes
    - `api_doc.md`, `sequence.md`, `process_flow.md`, etc.
    - Images for diagrams if applicable
4. Link outputs in the root `README.md` following the existing format
5. Submit a Pull Request with a brief explanation

### 3. Improve Existing Content
We welcome:
- Fixes to typos or formatting
- Updated reviews (tools evolve fast!)
- Standardizing filenames or folder structures

## 📁 Folder Structure

Each AI tool has its own folder and typically includes:
- `README.md` – Explanation and walkthrough
- Specific generated files (e.g. API docs, changelogs, diagrams)
- Screenshots or images where exports aren’t possible

Follow naming styles already in use (e.g. `process_flow.md`, `inline_comments.md`).
