# Security Policy

## Scope

This repository evaluates AI documentation tools and publishes non-executable outputs such as markdown, diagrams, and analysis. While there is no software deployed or executed directly from this repo, we welcome responsible reporting of:

- Security or privacy risks in included outputs (e.g. credentials, secrets)
- Potential license violations or data leaks from tested tools
- Vulnerabilities in documentation workflows (e.g. unsafe integrations)

## Reporting a Vulnerability

To report a potential issue, please create a **private GitHub issue**:

1. Go to [https://github.com/lextego/ai_tool_tests/issues](https://github.com/lextego/ai_tool_tests/issues)
2. Select the “Security Report” template (visible only to maintainers)
3. Provide as much detail as possible (tool name, concern, context)

Your report will be reviewed by LexTego Ltd maintainers. We will acknowledge receipt within **5 business days** and follow up as necessary.

## Disclosure

Please avoid publicly disclosing vulnerabilities before maintainers have reviewed the issue.
